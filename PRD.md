## 1. 产品概述

### 1.1 产品定位
小说阅读助理是一款面向深度阅读用户的 **All-in-One macOS 原生桌面应用**。它通过 LLM 自动解析小说内容，构建人物关系图谱、道具/功法/派系知识库，并结合本地 RAG（检索增强生成）技术，让用户能够以对话和可视化的方式探索小说世界。

### 1.2 核心价值
- **关系可视化**：将复杂的人物关系、门派恩怨以交互式图谱呈现，一眼看清江湖格局。
- **知识检索**：基于全文向量检索，快速定位功法描述、法宝出处、角色登场章节。
- **智能问答**：围绕小说内容做事实性问答，不依赖通用知识，只基于当前小说文本。
- **隐私优先**：Embedding 模型本地运行，小说内容不上传云端（除用户指定的 LLM API）。

### 1.3 目标用户
- 玄幻/武侠/仙侠类网文深度读者
- 需要梳理复杂关系网的重度小说爱好者
- 同人创作、二创内容创作者（需要快速查阅原设）

## 2. 功能需求

### 2.1 文档导入（FR-IMPORT）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-IMPORT-01 | 支持 `.txt` 导入 | P0 | 纯文本，自动识别编码（UTF-8/GBK/GB2312） |
| FR-IMPORT-02 | 支持 `.epub` 导入 | P0 | 提取章节结构、正文内容，保留目录层级 |
| FR-IMPORT-03 | 支持 `.pdf` 导入 | P1 | 提取文本，支持扫描版 PDF OCR（TBD，初版可仅支持文字版） |
| FR-IMPORT-04 | 批量导入与删除 | P1 | 支持多文件拖拽，已导入小说可删除并清理关联数据 |
| FR-IMPORT-05 | 章节自动切分 | P0 | 基于目录/章节标题正则/分页符自动拆分章节 |

### 2.2 知识图谱构建（FR-GRAPH）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-GRAPH-01 | 人物实体提取 | P0 | 自动识别角色名，去重（别名、道号、外号归一） |
| FR-GRAPH-02 | 人物关系抽取 | P0 | 识别师徒、仇敌、情侣、同门、亲属等关系 |
| FR-GRAPH-03 | 道具/法宝提取 | P0 | 提取武器、丹药、法宝、秘籍等道具实体 |
| FR-GRAPH-04 | 功法/技能提取 | P0 | 提取功法名称、所属派系、修炼条件、效果描述 |
| FR-GRAPH-05 | 派系/门派提取 | P0 | 提取宗门、家族、势力范围、阵营信息 |
| FR-GRAPH-06 | 实体关系编辑 | P1 | 用户可手动增删实体、修正关系、添加别名 |
| FR-GRAPH-07 | 图谱持久化 | P0 | 图数据库存储，支持增量更新（重新导入时合并而非全量重建） |

### 2.3 RAG 全文检索（FR-RAG）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-RAG-01 | 本地 Embedding | P0 | 使用本地模型将章节/段落编码为向量 |
| FR-RAG-02 | 向量存储 | P0 | 本地向量数据库持久化存储 |
| FR-RAG-03 | 语义检索 | P0 | 支持自然语言查询，返回最相关段落（Top-K） |
| FR-RAG-04 | 混合检索 | P1 | 向量检索 + 关键词检索（BM25）混合排序 |
| FR-RAG-05 | 检索结果溯源 | P0 | 返回结果标注出处章节，支持一键跳转原文 |
| FR-RAG-06 | 多小说隔离 | P0 | 每本小说独立向量空间和图谱空间 |

### 2.4 可视化展示（FR-VIS）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-VIS-01 | 人物关系图谱 | P0 | 力导向图展示人物节点与关系边，支持缩放/拖拽 |
| FR-VIS-02 | 派系势力图 | P0 | 按门派聚类展示，显示派系人数、核心人物 |
| FR-VIS-03 | 人物详情面板 | P0 | 点击人物节点展示：所属派系、掌握功法、持有道具、关系列表 |
| FR-VIS-04 | 功法/道具详情 | P0 | 展示描述、关联人物、首次出场章节 |
| FR-VIS-05 | 时间线视图 | P2 | 按章节/事件排序展示人物登场和重要剧情节点 |
| FR-VIS-06 | 图谱筛选 | P1 | 按派系、关系类型、章节范围筛选节点和边 |

### 2.5 智能问答（FR-CHAT）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-CHAT-01 | 小说问答 | P0 | 基于 RAG 检索上下文 + LLM 生成答案，限定仅使用当前小说内容 |
| FR-CHAT-02 | 人物/功法查询 | P0 | "某某某的功法有哪些？" → 结合图谱+RAG回答 |
| FR-CHAT-03 | 原文引用 | P0 | 答案中引用原文段落，标注章节 |
| FR-CHAT-04 | 多轮对话 | P1 | 支持上下文关联的多轮追问 |
| FR-CHAT-05 | 会话历史 | P1 | 保存问答历史，支持查看和删除 |

### 2.6 LLM 配置（FR-LLM）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| FR-LLM-01 | OpenAI 兼容接口 | P0 | 仅需配置 `base_url`, `model_name`, `api_key` |
| FR-LLM-02 | 模型切换 | P1 | 支持保存多组配置，快速切换 |
| FR-LLM-03 | 解析与对话分离 | P2 | 可分别指定解析用模型和对话用模型（如解析用强模型，对话用快模型） |

## 3. 非功能需求

### 3.1 依赖约束（NFR-DEP）
- **NFR-DEP-01**：对外部服务的依赖仅限 LLM API（`base_url` + `model_name` + `api_key`）。
- **NFR-DEP-02**：Embedding 推理必须在本地完成，禁止将小说原文发送到外部 Embedding 服务。
- **NFR-DEP-03**：所有数据（图谱、向量、解析结果、配置）本地存储，不依赖远程数据库。

### 3.2 性能要求（NFR-PERF）
- **NFR-PERF-01**：100万字小说导入+解析时间 < 10分钟（M1 Pro 级别机器）。
- **NFR-PERF-02**：图谱可视化支持 500+ 节点流畅交互（60fps）。
- **NFR-PERF-03**：RAG 检索响应时间 < 2秒。

### 3.3 可维护性（NFR-MAINT）
- **NFR-MAINT-01**： macOS 14+ 支持，Apple Silicon / Intel 双架构。
- **NFR-MAINT-02**：应用包体积控制在 200MB 以内（不含本地模型）。
- **NFR-MAINT-03**：本地模型（Embedding）可自动下载/更新，也可手动指定路径。

### 3.4 数据安全（NFR-SEC）
- **NFR-SEC-01**：小说原文、图谱数据、向量索引均本地加密存储（AES-256）。
- **NFR-SEC-02**：LLM API Key 存储在 macOS Keychain 中。

## 4. 用户故事

### US-01：导入与自动解析
> 作为读者，我下载了一本 500 万字的修仙小说 EPUB，想把它导入应用。我拖拽文件到应用窗口，应用自动提取章节、识别人物、门派、功法，并在 5 分钟后告诉我解析完成，可以查看人物关系图。

### US-02：探索人物关系
> 我想搞清楚主角和各大门派的关系。我打开"人物关系图谱"，看到主角位于中心，各门派人物围绕四周，红色边表示仇敌，绿色表示盟友。我双击"青云门"节点，图谱自动聚焦到青云门内部的人物关系。

### US-03：查询功法信息
> 我忘记"九转玄功"是谁修炼的了。我在搜索框输入"九转玄功是谁的功法"，应用通过 RAG 检索相关段落，结合图谱中的"功法-人物"关系，给出答案："九转玄功由主角李逍遥在第三卷第 45 章开始修炼，源自上古遗迹……"并附上原文引用。

### US-04：修正图谱
> 应用把"小舞"和"舞儿"识别成了两个人。我在人物列表中找到这两个实体，选择"合并实体"，指定主名为"小舞"，别名为"舞儿"，图谱立即更新，所有关系自动合并。

### US-05：更换模型
> 我想用本地部署的 Ollama 做问答。我进入设置，填写 `base_url=http://localhost:11434/v1`，`model_name=llama3:8b`，点击测试连接，成功后即可使用。

## 6. 里程碑规划

| 阶段 | 周期 | 交付物 |
|---|---|---|
| M1 | 2周 | 基础框架：macOS 应用壳、txt/epub/pdf 导入、章节拆分、LLM 配置 |
| M2 | 3周 | 图谱系统：实体/关系提取、图数据库存储、人物关系可视化 |

## 7. 术语表

| 术语 | 说明 |
|---|---|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Embedding | 将文本编码为向量的过程 |
| 图谱 / Graph | 由实体（节点）和关系（边）组成的知识网络 |
| 实体 | 小说中的具体对象：人物、道具、功法、派系等 |
| 分块 / Chunking | 将长文本切分为适合向量检索的短段落 |
